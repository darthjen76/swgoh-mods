## Architecture

**Stack:** React 19 + TypeScript + Vite + Tailwind CSS v4 + Zustand + TanStack Query. No backend — fully static SPA deployed on Vercel. All persistence is `localStorage` via Zustand `persist`.

**Data sources:**
- **Player roster + mods** → [SWGOH Comlink](https://swgoh-comlink-latest-wuy6.onrender.com) (official game API, no auth needed). Deployed on Render free tier — cold starts take ~30s.
- **Mod recommendations** → `public/recommendations.json` (static, fetched from GitHub raw on manual update). Generated by `scripts/generate-recommendations.mjs` which queries Comlink's GAC Kyber leaderboard.

**Why no swgoh.gg API:** swgoh.gg is Cloudflare-protected and blocks datacenter IPs (Vercel). CORS also blocks direct browser calls. Comlink bypasses both issues.

## Comlink data format

Comlink returns raw game data, which differs from swgoh.gg:

- **Mod `definitionId`** encodes set/slot/rarity as `"{set}{tierLetter}{slot}"` — e.g. `"4E2"` = Speed set (4), 5-dot (E), Arrow slot (2). Parsed in `useSwgoh.ts → parseModDefinitionId()`.
- **Stat IDs** are numeric (`5` = Speed, `41` = Offense %, etc.) — see `STAT_NAMES` in `src/types/swgoh.ts`.
- **Percentage stats** (IDs: 17, 18, 19, 41, 42, 53, 54, 55, 56) come in as decimals from Comlink (e.g. `0.054` = 5.4%) — multiplied by 100 in `makeModStat()`.
- **Character `definitionId`** format is `"BASE_ID:STAR_LEVEL"` — base_id is extracted by splitting on `:`.
- **`combatType`** field: `1` = character, `2` = ship. Filter on `1` to exclude ships.

## Recommendations system

`src/data/recommendations.ts` uses two layers:
1. **Role templates** (`ROLE_TEMPLATES`) — 9 roles (e.g. `gl_attacker`, `speed_support`, `tank`) with sets/primaries/avg_speed
2. **Character → role mapping** (`CHARACTER_ROLES`) — maps SWGOH base_ids to roles
3. **Specific overrides** (`SPECIFIC`) — full custom recommendations for ~20 meta characters (SLKR, Rey, JML, CLS, Darth Revan, etc.)

`getRecommendation(base_id)` checks overrides first, then role mapping, then defaults to `speed_attacker`.

The "Update" button in Settings fetches `public/recommendations.json` from the GitHub raw URL. To publish new recommendations: update the JSON file and push to main.

## Mod domain knowledge

- **Slots:** 1=Square/Transmitter, 2=Arrow/Receiver, 3=Diamond/Processor, 4=Triangle/Holo-Array, 5=Circle/Multiplexer, 6=Cross/Data-Bus
- **Sets:** 1=Health, 2=Offense, 3=Defense, 4=Speed, 5=CritChance, 6=CritDamage, 7=Potency, 8=Tenacity
- **Slicing** = upgrading mod quality tier (grey→green→blue→purple→gold). Triggered by stat threshold conditions stored in `sliceConditions` in the Zustand store.
- Speed is the dominant secondary stat — always highlighted in blue in the UI.
